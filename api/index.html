    API Reference¶

    Auto-generated from module docstrings by mkdocstrings.

    The reference is organized per top-level composite, mirroring the session object's layout:

    Page Access Purpose
    Session apeGmsh(...) Top-level session lifecycle and composite wiring
    Model g.model OCC geometry (geometry, boolean, transforms, io, queries)
    Parts g.parts Part registry and assembly bookkeeping
    Mesh g.mesh Meshing (generation, sizing, field, structured, editing, queries, partitioning)
    Constraints g.constraints Solver-agnostic constraint defs, records, resolver
    Loads g.loads Load defs, records, resolver
    Masses g.masses Mass defs, records, resolver
    FEM Broker g.mesh.queries.get_fem_data() FEMData snapshot container
    OpenSees g.opensees OpenSees bridge (materials, elements, ingest, inspect, export)
    Results Results Post-processing container
    Viewers Qt/PyVista model and mesh viewers
    Viz g.plot, etc. Matplotlib, selection, VTK export

    Note

    Private members (names starting with _) are filtered out by default. Sub-composite implementation modules (_mesh_generation, _model_geometry, etc.) are documented under their parent composite page.
