# ADR 0055 — Staged-model H5 archival

**Status:** Proposed (2026-06-07). **Builds on** ADR 0021 (lineage chain), ADR 0023 (two-version reader window), ADR 0034 (staged analysis), ADR 0052 (staged reference position / HOLD). **Related to** ADR 0038 (compose model composition, §169 stages = FILTER+warn), ADR 0048 (inferred ndf round-trip). **Closes** the SSI-2 deferral "staged H5 archival" (`staged-analysis.md:570`, `_DEFERRED.md:146-151`) and **inverts** the `test_h5_staged_fail_loud.py` guard.

## Context

A staged-analysis model (`ops.stage(...)`, ADR 0034) cannot be archived to `model.h5` today. `apeSees.h5()` fails loud **before** building: it raises `NotImplementedError` when `self._stage_records` is non-empty (`apesees.py:5493`) or when `self._initial_stress_records` is non-empty (`apesees.py:5502`). These are two independent guards on two independent buckets. The docstring `Raises` block (`apesees.py:5471-5487`) states the hazard precisely: a silent `h5()` would round-trip a staged model to a **flat** model — dropping every stage's analysis-chain rebinding, per-stage `activated_pgs`, per-stage initial-stress records, and the per-stage `analyze` loop. The guard exists because the H5 emitter's 10 staged Protocol methods (`stage_open`/`stage_close`/`domain_change`/`addToParameter`/`step_hook_ramp`/`set_time`/`set_creep`/`reset`/`remove_sp`/`remove_element`, `h5.py:1327-1374`) are all `del`-and-return no-ops; without them no staged byte payload exists.

This is the last large fail-loud gap in the persistence story. Everything else round-trips: the flat read path (`OpenSeesModel.from_h5`, `opensees_model.py:167-330`) and the emitter-agnostic flat replay (`_replay_into`, `compose.py:247-459`) reproduce non-staged decks across tcl/py/h5. The deferred-zone comment (`h5.py:1302-1325`) already sketches the target shape (a `/opensees/stages/stage_NN` zone) but its version targets are **stale** (it cites `2.11.0 → 2.12.0`; the SSI-2.E comment cites `2.7.0 → 2.8.0`; the live constant is `2.15.0`, `h5.py:298`). A second call site is coupled to the same gap: `ops.domain_capture` passes `bridge=None` for staged builds (`apesees.py:4549-4552`), losing the ndf sidecar round-trip "until H5 staged-archival lands."

What makes this hard is the per-stage state (`StageRecord`, `build.py:631`): ~25 fields including **7 analysis-chain primitive references** (not copies — they live on `bridge._primitives` and resolve tags via the topological pass), stage-owned pattern/recorder/damping primitives with cross-stage `id()`-sharing and global-skip claim sets (`_stage_claimed_pattern_ids`/`_stage_claimed_recorder_ids`), `stage_constraint_records` carrying **resolved** master/slave node tags routed *out* of `fem.*.constraints` (`build.py:664-674`), and a global `fem_eid→ops_tag` map allocated once pre-stage and shared across every stage/rank that `addToParameter` and Element-recorder fan-out index by. The full staged emit also has **two materially different paths** — flat (`_emit_stages_flat`, `apesees.py:1359-1492`) and partitioned (`_emit_stages_partitioned`, `apesees.py:2068-2501`) — that differ in bracketing, `domain_change` count, pattern fan-out, and (critically) the partitioned path **raises** on any HOLD support (`apesees.py:2144-2152`).

Why now: the SSI-2 phases declared this work and left the read trio half-built — `ParameterRecord`/`AddToParameterRecord`/`StepHookRampRecord` already exist in `typed_records.py:588-620` precisely so "the future schema bump is mechanical." The global-initial-stress bucket is fully declarative and cheap; lifting it proves the schema/hash/version machinery end-to-end on a small surface before the expensive staged surface is built.

## Decision

Archive authored staged state as a **structured `/opensees/stages` tree of declarative, pre-resolve records** (mirroring the `regions`/`dampings`/`partitions` writers — no JSON, queryable compound tables), plus a top-level `/opensees/initial_stress` table for the separate global bucket. On read, rehydrate read-side dataclasses and re-drive the documented per-stage emit sequence through an emitter-agnostic `_replay_staged_into`. The zone folds into `model_hash` (it is authored model definition, same class as `dampings`/`nodes_ndf`).

Because this is the largest writer+reader surface in the persistence layer and the staged feature's canonical use case (partitioned SSI with HOLD supports) **cannot** round-trip until partitioned-HOLD lands, the work is **sliced** (see Phasing). The design below is the full target; the first shippable slice is Phase 1 (global initial-stress only), and each later slice keeps the remainder fail-loud.

### Store-and-echo: declarative, pre-resolve, never build-time-allocated

Governing rule for every byte that enters the hashed `/opensees/stages`: persist the **pre-resolve declarative form**, never a value that is non-deterministic between write and `from_h5 → to_h5` re-emit. Concretely:

- Region references by **name** (V3-mangled), never by tag — regions are excluded from `model_hash` and re-allocated at emit; a stored region tag goes stale.
- Initial-stress as the `InitialStressRecord` field set (`name`, `pg` XOR `elements`, `sigma_xx/yy/zz`, `ramp_steps`, `lambda_install`) — **never** rendered `step_hook_ramp` proc text. Parameter tags and `StepHookRampRecord` targets are allocated at emit; replay re-runs `emit_initial_stress_global` to regenerate them byte-identically (the proc body is a pure function of `(name, targets, n_steps_to_full, phase)`, `tcl.py:445-504`).
- Analysis chain **by value**: each stage's 7 chain directives serialized as `(token, *args)` attr bags inline under the stage group (the `_write_analysis` shape, `h5.py:1380-1409`), **not** as a tag reference. The global `/opensees/analysis` chain is skipped for staged builds, so there is no tag to point at. Value-copy per stage is semantically equivalent because emit re-emits the chain line once per referencing stage anyway.

This rule is what makes the ADR 0023 two-version window meaningful for staged files: the window gates layout only; declarative store-and-echo is the actual round-trip-fidelity guarantee.

### Owned topology: persist resolved tags, do NOT re-derive ownership (adversarial fix)

`compute_stage_ownership` (`build.py:2807-2842`) keys on `id(Element_primitive)` and matches each primitive's `pg=` against `activated_pgs`. The read side carries **resolved `ElementRecord`s (tag-only, connectivity dropped)** — there is no element-spec→PG association to re-derive from. Therefore replay must **not** call `compute_stage_ownership`. Instead, each `stage_NNN` group persists the **owned element tags and owned node tags** as explicit `int64` datasets (exactly like partitions' `element_ids`/`node_ids`, `h5.py:2338-2340`). Replay emits those exact tags inside the stage block by value. `activated_pgs` is retained **only** as a manifest label (graft from the deck-blob strategy: a fast `(name, n_increments, dt, activated_pgs)` index for the `.stages()` accessor and a future Stage discriminator), never as a replay input.

### Global element-tag map: persist, and re-validate on replay (adversarial fix)

`addToParameter` and Element-recorder fan-out index by the shared `fem_eid→ops_tag` map (`apesees.py:1625-1668`), allocated once by `allocate_element_tags` pre-stage. Persist it as a top-level `/opensees/element_tag_map` (`int64` pair dataset); it folds into `model_hash` (authored identity). On replay, **re-run `allocate_element_tags` over the rehydrated element pool and assert it reproduces the persisted map** — fail loud on mismatch rather than blindly re-applying. This catches allocation-order drift at read time instead of silently misrouting `addToParameter`/recorder targets. `emit_initial_stress_addtoparameter`'s existing fail-loud guard (`build.py:3005`, raises when an eid is absent from the map) is preserved.

### Resolved stage-constraint node tags and the build-order hash caveat (adversarial fix)

`stage_constraint_records` carry resolved `master_node`/`slave_node`/`slave_nodes` into the global node pool and were deliberately routed out of `fem.*.constraints` (`_constraints.py:83/175`), so they cannot be recovered from the FEM broker. We freeze the resolved tags **and** rely on the archived global node numbering being preserved on replay (valid only while the node pool is immutable; any renumber invalidates them — documented, consistent with INV-5, `opensees_model.py:50-58`). This makes `from_h5 → to_h5` byte-stable (frozen values re-emit identically) but means staged `model_hash` is **allocation-order-sensitive**: a fresh `apeSees(fem).run()` of the same model may not match the archived hash if `allocate_element_tags`/node ordering changes. We commit to identity model **(b)**: staged `model_hash` is a strictly stronger INV-5 caveat (round-trip-stable, not build-to-build-stable). A regression test asserts two *fresh* builds of the same staged model produce the same `model_hash`; if that test cannot pass on a given allocator, declarative-by-PG re-resolution of constraints (option (a)) becomes mandatory for that record kind. We do not pre-emptively pay for (a).

### Group shape

Parent `/opensees/stages` with an `n_stages` `int64` count attr. `_write_stages(f)` is called **last** in `write_opensees_into` (after `_write_analysis`, `h5.py:1462-1475`) so stages land inside the `model_hash`-hashed `/opensees` group and reference already-written materials/sections/series/dampings/elements. **Early-return before `create_group('stages')` when the buffer is empty** (regions/dampings/partitions precedent) — vanilla `/opensees` stays byte-identical, `model_hash` unchanged, pinned by `test_h5_non_staged_smoke_still_writes`.

Source of truth: capture `BuiltModel.stage_records` via a `set_stage_records(...)` side-channel that `BuiltModel.emit` calls before driving the staged sequence (the 10 no-op Protocol methods stay no-ops, kept only for direct Protocol-conformance tests). The writer resolves each primitive reference through `bm`'s tag resolver at capture time to `(token, args, tag)` — it must have access to the resolver, not just the record tuple.

Per `stage_{idx:03d}` group (zero-padded so name-sorted canonical order == registration order, load-bearing for cross-stage tag identity and `loadConst` boundaries):
- Scalar attrs via `_set_attr`: `name`, `n_increments`. **Tri-state float/bool fields** (`dt`, `set_time`, `set_creep_on`, `pre_analyze_reset`) are **presence-flagged** (adversarial fix): `None`/never-called must map to *omit the call*, not to `setTime 0.0`/`setCreep 0`/`reset`. Use the `_write_param_array` NaN-sentinel idiom for floats (`dt`, `set_time`); a `*_present` `int64` companion for the booleans (`set_creep_on`, `pre_analyze_reset`). A round-trip test asserts the emitted line count differs across {never-set, value-0, value-nonzero}.
- `activated_pgs` as a **deterministically-ordered** vlen-string dataset (write the `StageRecord` tuple verbatim — never `sorted()`-from-a-`set()`, which would drift the hash).
- `owned_element_ids`, `owned_node_ids` as `int64` datasets (see above).
- Inline analysis chain as a `(token, *args)` attr bag scoped to the group (see store-and-echo).
- Structured sub-sub-tables, each a compound dataset or `_write_param_array` arg-tail (never JSON): `fix_records` (pg vlen-str XOR node-ids `int64`, dofs), `mass_records` (+ `overwrite` bool), `region_records` (name vlen-str), `recorder_specs`, `rayleigh_records` (coefficients `float64` + `on` vlen-str), `damping_attach_records` (damping handle + `on`), `pattern_specs` (**fully-resolved**, see below), `support_records` (+ `support_pattern` presence flag bound to the shared `_hold_series` handle), `stage_constraint_records` (resolved `ConstraintRecord` compound, reusing `_write_constraints_*`), `remove_sp_records`/`remove_element_records`, and a per-stage `initial_stress` sub-table (`InitialStressRecord` compound).

**Per-stage constraint compound width** (adversarial fix): the `dofs` sub-shape for *every* stage's `stage_constraint_records` is pinned to the **global `self._ndf` envelope** (which round-trips via the `model(ndm,ndf)` directive), **never** a stage-local `max(len(dofs))`. Otherwise a stage whose dofs are narrower than the global ndf produces a different compound `dtype` on `from_h5 → to_h5` and `model_hash` silently drifts (`_dtype_tag` is hashed, `lineage.py:241-243`). A round-trip hash test builds exactly this narrower-than-ndf case.

### Stage patterns: fully resolved, replay verbatim (adversarial fix)

The flat path does **not** re-resolve patterns at replay — `_replay_into` (`compose.py:411-422`) emits already-resolved `PatternRecord`s (concrete node tag + force vector); the `pg=` fan-out and `from_model(case)` expansion (`emit_pattern_spec`, `build.py:1437-1560`) ran once at write time. Staged patterns mirror this exactly: serialize each stage pattern as a **fully-resolved record** (pattern token, series handle, concrete load/sp lines already expanded). Replay emits the frozen lines verbatim; it **never** re-runs `emit_pattern_spec` at replay. This both matches `_replay_into` and keeps the hashed bytes deterministic (resolved, echoed-not-recomputed). The `from_model(case)`-by-name external cross-reference is resolved at write time, not persisted as a dangling case name.

### Replay model: inline emit, no claim sets (scope decision)

`_replay_staged_into` is a new free function, sibling of `_replay_into`. The read-side broker has **no live global pattern/recorder pass**, so there is nothing to double-emit and **no claim sets are rehydrated** — `_stage_claimed_pattern_ids`/`_stage_claimed_recorder_ids` and their plumbing are **dropped from the read surface entirely** (forced scope decision; the by-handle/claim-set alternative is rejected). Replay emits stage patterns/recorders/chain **inline per stage**. The `_hold_series` singleton is re-created once on replay and shared across all stages' `support_pattern`.

`build()` (`opensees_model.py:435-490`) and both H5 populate paths (`_populate_emitter` 846-918, `_populate_emitter_h5` 976-1029) detect `_stages` non-empty and route to `_replay_staged_into`. It first emits the global (non-staged) prefix (topology/materials/sections) via the existing flat path, then per `StageRecordRO` re-drives the documented order (`staged-analysis.md:166-235`): `stage_open(name)` → `set_time`/`set_creep` → owned nodes/elements (by stored tag) → `remove_sp`/`remove_element` → `fix`/`mass`/`region` → stage MP constraints → support HOLD → `domain_change` → `rayleigh`/`damping_attach` → `initial_stress` → inline analysis chain → stage patterns → recorders → `pre_analyze_reset` → `analyze(n_increments, dt)` → `stage_close`.

**Replay must branch on `is_partitioned(fem)`** exactly as `BuiltModel.emit` does (adversarial fix): the flat and partitioned emit *shapes* differ (per-rank `partition_open`/`close` bracketing, single global `domain_change`, per-rank pattern fan-out). The partition layout is recoverable from the neutral `/partitions` zone. Until a partitioned staged replay exists, the guard lift is scoped to **non-partitioned staged decks only** (see Phasing) — `ops.h5` stays fail-loud for partitioned staged builds, not merely partitioned-HOLD.

### Flat global initial-stress wiring (adversarial fix — the cheap bucket has a hook-wrap trap)

The global initial-stress bucket is **not** static-table-only. For a non-staged model with `initial_stress` but no stages, the deck is `parameter` decls + ramp proc + `addToParameter` + a **hook-wrapped global `analyze`** (a for-loop, not a bare `analyze N`) — the wrapping is driven by `emitter._step_hooks_registered`, a side effect of `step_hook_ramp`. The flat `_replay_into` today emits `analyze_call` as a bare `analyze` with no preceding `step_hook_ramp`, so a naive global-initial-stress round-trip would declare the ramp procs but **never invoke them** — in-situ stress installed as dead code, silently. Therefore `_replay_into` (the **flat** path) gains an `initial_stress=` parameter and emits `emit_initial_stress_global` + `emit_initial_stress_addtoparameter` **before** `_replay_analysis_chain`/`analyze_call`, so `_step_hooks_registered` flips and the global `analyze` re-wraps. `OpenSeesModel` gains `_initial_stress` + `.initial_stress()`, threaded through both `_populate_emitter` and `_populate_emitter_h5`. The two guards are **not** lifted "together" — the global bucket has its own flat-replay wiring distinct from the staged path.

### Read side

Add `StageRecordRO` and `InitialStressRecordRO` to `typed_records.py` (tag/value form, not primitive references): the 7 chain directives as `(token, args)` tuples; pools as read-side typed records; initial-stress as the existing `ParameterRecord`/`AddToParameterRecord`/`StepHookRampRecord` feedstock (`typed_records.py:588-620`). Add `_stages: tuple[StageRecordRO, ...]` and `_initial_stress: tuple[InitialStressRecordRO, ...]` fields to the `OpenSeesModel` dataclass (`opensees_model.py:128-161`) plus `.stages()`/`.initial_stress()` accessors. `cls._load_stages(model_group)` walks `/opensees/stages/stage_NNN` in zero-padded name order (mirrors `_load_bcs`/`_load_analysis`). `from_compose_buffers` (332-406) also populates `_stages` from emitter buffers. Older files lacking `/opensees/stages` → empty tuple → flat replay (the `nodes_ndf` legacy-absence pattern, `opensees_model.py:284-297`). `ModelData.from_h5` already probes children with `in` (`model_data.py:524-566`) so it tolerates the new group unchanged.

### Read-time self-consistency validation (graft from the call-journal strategy)

On load, `_load_stages` asserts structural well-formedness and **fails loud** on a malformed archive: every `stage_open` is paired with a `stage_close`, and the between-stage `loadConst -time 0.0` + `wipeAnalysis` reset steps are reconstructable. This is the read-side analog of the write-side fail-loud invariant — a corrupt or partially-written stages zone never replays silently as a different program.

### Schema / hash / version

Additive **MINOR** bump `SCHEMA_VERSION 2.15.0 → 2.16.0` (`h5.py:298`, single source of truth via `schema_version.py`). Add a 2.16.0 History bullet describing `/opensees/stages`, `/opensees/initial_stress`, and `/opensees/element_tag_map`, that all three fold into `model_hash`, and the ADR 0023 incantation. Reconcile the **three stale** version comments to 2.16.0: `h5.py:1315` (`2.11.0 → 2.12.0`), `h5.py:1357-1358` (`2.7.0 → 2.8.0`), and the `apesees.py` guard comment / `test_h5_staged_fail_loud.py` docstring.

`MODEL_HASH_EXCLUDED_CHILDREN` (`lineage.py:91-93`) is **unchanged** — `stages`, `initial_stress`, and `element_tag_map` are authored model state (same decision as `dampings`/`nodes_ndf`) and must fold in. `_canonical_walk` honors exclusion only at `is_root`, so the new children fold automatically. Float determinism is sound on this path (`_set_attr`/`_write_param_array` → `np.float64.tobytes()`, no decimal formatting, `lineage.py:298-299`).

**Two-version window is a producer-side hard floor, stated honestly** (adversarial fix, verified against `schema_version.py:289-305`): once `SCHEMA_VERSION` flips to 2.16.0, *every* file the writer stamps is 2.16.0 — including vanilla non-staged files (the stamp is the module constant, not content-driven). A still-deployed 2.15.0 reader hits `file.minor(16) > supported_high(15)` → `SchemaVersionError` and **refuses** the file. The window only lets a 2.16 reader open 2.15 files, **not** the reverse. The 2.16.0 History bullet and this ADR state this as a hard floor ("all readers must be ≥ 2.16 to open any file written after this lands"), **not** as "old 2.15 readers ignore the new group" (which is false under the refuse-newer-minor branch). The prior `dampings` 2.14→2.15 bullet (`h5.py:296`) makes the same overclaim and is corrected in the same pass.

### Compose

`g.compose()` of a staged module = FILTER + `ComposeFilterWarning`, per the already-accepted ADR 0038:169. The compose pipeline reads only the neutral FEMData zone (`_compose.py:1106-1135`); it never copies `/opensees/` into the host, so `/opensees/stages` is **dropped by construction** — no `master_node`/`slave_node` tag-rewrite plumbing touches stage records. The warning probe (`_compose.py:2939-2953`) already reads `stages in ops_zone`; it fires the moment the group exists. Surface it in `g.compose_inspect`'s filtered-audit (0038:195-196). This was a paper decision until the structured-record strategy made staged H5 reachable.

### Guard lift

Lift the two `NotImplementedError` branches (`apesees.py:5493`, `5502`) and the `Raises` docstring (5471-5487) **independently and only** when each bucket's writer + RO reader + replay + accessor land and round-trip byte-stable. Until full parity, fail loud on any unhandled remainder — in particular `_emit_stages_partitioned` keeps **raising on `support_records`** (`apesees.py:2144-2152`), and partitioned staged replay stays fail-loud until its slice lands. LiveOpsEmitter staged stays fail-loud (its `stage_open`/`close` RAISE — separate ADR 0034 deferral).

## Consequences

**Positive**
- Staged and global-initial-stress models become archivable, queryable (`.stages()`/`.initial_stress()`), and lineage-folded; the last large fail-loud persistence gap closes (incrementally).
- Declarative store-and-echo makes the ADR 0023 window actually protect staged fidelity, and makes `from_h5 → to_h5` byte-stable.
- Inline-emit replay with **no claim sets** is a genuine simplification of the read surface vs. the bridge's global-pass dedup machinery.
- The recording-emitter test oracle (graft from the call-journal strategy) and the read-time self-consistency check make order/pairing regressions fail loud rather than silently misroute.

**Negative / accepted**
- Largest writer+reader surface in the persistence layer; most byte-canonical drift surface. Mitigated by phasing and the `model_hash`-stability regression test (the single most important guard).
- Value-copy of the analysis chain discards `id()`-sharing of chain primitives across stages. Semantically equivalent (chain re-emits per referencing stage anyway) but **forecloses** a future "shared mutable stage primitive" feature without a schema change. Accepted.
- Staged `model_hash` is **allocation-order-sensitive** (identity model (b)): round-trip-stable, not guaranteed build-to-build-stable. Stronger INV-5 caveat, documented; guarded by the two-fresh-builds test.
- Resolved stage-constraint node tags and the persisted `element_tag_map` **pin the node/element pool** — any renumber invalidates the archive. Re-validate-on-replay catches drift loudly; it does not repair it.

**Stays deferred (explicitly scoped out)**
- **Partitioned staged replay** — flat-only first; `ops.h5` stays fail-loud for partitioned staged builds.
- **Partitioned HOLD supports** — `_emit_stages_partitioned` raises today; replay keeps it fail-loud. The canonical Cerro-Lindo SSI-with-HOLD deck therefore does **not** round-trip until both partitioned legs land — this is the explicit reason not to front-load the full structured surface.
- **Per-stage modal damping** — not staged (`build.py:719`); must keep raising, never silently persist.
- **Live staged emit** — `LiveOpsEmitter.stage_open/close` RAISE (ADR 0034 deferral).
- **Results-zone Stage discriminator / per-stage viewer slab** — `_DEFERRED.md:167-170`, orthogonal RESULTS-zone concern. The persisted `activated_pgs`/stage names are the future feedstock.
- **`g.compose()` FILTER+warn implementation** and **`ops.domain_capture` `bridge=None` ndf revisit** — each is a separable PR gated on Phase 1 landing (an ADR 0038 follow-through and an ADR 0048 round-trip concern respectively), **not** bundled into the writer PR.

## Phasing

Sequence value; keep the remainder fail-loud at every step. The two buckets are guarded independently and **lifted independently**.

1. **Global initial-stress archival** → verify: `from_h5 → to_h5` byte-stable; rendered `step_hook_ramp` proc body string-equal across original-Tcl vs round-trip-Tcl; the **flat** `_replay_into` `initial_stress=` wiring makes the global `analyze` re-wrap (recording-oracle asserts `step_hook_ramp` precedes `analyze`). Smallest defensible slice — read trio already exists (`typed_records.py:588-620`), fully declarative, no primitive graph, no element-tag map, lifts the `5502` guard alone. Schema bumps 2.15.0 → 2.16.0 here; the three stale comments reconcile here.
2. **Non-partitioned staged structure** → verify: deck structural/line equality (NOT raw-tag per INV-5) of `stage_open`/`close` delimiters, per-stage chain lines, owned topology, initial-stress procs, hook-wrapped analyze loops, between-stage `loadConst -time 0.0` + `wipeAnalysis`; `model_hash`-stability (staged hash ≠ one-stage-removed hash; `from_h5 → to_h5` reproduces identical hash; two fresh builds match); the narrower-than-ndf constraint-width hash case; `element_tag_map` re-validation fails loud on injected drift. Lifts the `5493` guard for non-partitioned decks only. **Invert `test_h5_staged_fail_loud.py`**: delete the two raise-asserting cases, keep the vanilla smoke case (+ assert no `/opensees/stages` key after a vanilla write). Recording-emitter oracle becomes a **mandatory** round-trip gate (assert replay call stream == original `BuiltModel.emit` call stream, including relative order of `step_hook_ramp` vs `analyze`), for both flat and staged fixtures.
3. **Compose FILTER+warn** (separate PR, gated on Phase 1) → verify: a staged module contributes **zero** `/opensees/stages` bytes to the composed file and emits exactly one `ComposeFilterWarning` surfaced in `g.compose_inspect`, with the drop occurring before any node-tag import.
4. **`ops.domain_capture` ndf-sidecar revisit** (separate PR, ADR 0048 concern) → verify: the ndf sidecar round-trips for staged builds once `bridge=None` degrade is removed.
5. **Partitioned staged replay** (deferred, demand-gated) → verify: `partition_open`/`close` bracketing, single global `domain_change`, per-rank pattern fan-out reproduced; partitioned-HOLD remains a separate fail-loud leg until its own slice.

Phases 2–5 land only after a concrete user need for staged H5 (beyond the existing tcl/py emit paths) is named; until then the staged guard stays loud past Phase 1. Each phase lands with its own round-trip + hash-stability tests; no bucket's delivery is coupled to a costlier bucket's completion (avoiding the #555/#556 coupled-change orphan history).

## Open questions / resolved trade-offs

- **Resolved: identity model for `model_hash`.** Chose (b) round-trip-stable over (a) build-to-build-stable, freezing resolved constraint node tags + the `element_tag_map`. (a) (declarative-by-PG constraint re-resolution) is held in reserve, mandatory only if the two-fresh-builds test fails on a given allocator. Rationale: (b) is cheaper and the documented INV-5 envelope already tolerates tag divergence; we do not pre-emptively pay for (a).
- **Resolved: replay model.** Inline per-stage emit with **no claim sets** (the read broker has no global pattern/recorder pass to dedup against). The by-handle + claim-set rehydration alternative is rejected; the dual presentation in the source design is collapsed to this single decision.
